\section*{System model}

\paragraph{Network model.}
Pulsar assumes synchronous broadcast within the round window: all
honest parties receive all honest broadcast messages within a bounded
$\Delta$. The reference deployment uses authenticated, eventually-
delivered broadcast channels; authenticated, encrypted point-to-point
channels for share / blind delivery; and an authenticated,
append-only public bulletin (consensus-anchored) for DKG commitments
and reshare records.

\paragraph{Setup assumptions.}
Each party advertises a long-term ML-DSA-65 verification key on the
public bulletin (per HIP-0077 validator-set records). A shared
randomness oracle (the Lux consensus randomness beacon) supplies
the per-reshare quorum seed.

\paragraph{Trust model.}
Up to $t-1$ corrupted parties, static (the MPTC requirement).
Liveness requires an honest majority of $n - t + 1$ available
parties. The proactive resharing protocol handles the HJKY97 mobile
adversary by re-randomising shares at each epoch.

\paragraph{Abort model.}
Identifiable abort: every protocol stage emits a signed complaint
when a deviation is detected, naming the deviating party. The
consensus layer consumes these complaints for slashing.

\paragraph{Preprocessing and rotation.}
DKG runs once per validator-set epoch. Reshare runs at every
epoch boundary; typical cadence is hours to days. Sign runs
per-message with no preprocessing requirement (the offline / online
split is optional). Members join and leave at epoch boundaries
subject to the $(n', t')$ constraint.

\paragraph{Monitoring.}
Public artifacts safe for observability: $\mathit{pk}$, commitments,
signed complaints, signature outputs, beacon values, identity-key
signatures. Private artifacts that MUST NOT be logged: shares,
blinds, masks, samples, PRNG seeds, per-round derived state,
norm-check intermediate values.

\paragraph{Deployment surface.}
The reference implementation is software (Go). HSM-backed share
custody is a deployed-binary concern handled by the operator
hardening checklist in \texttt{docs/deployment.md}.
